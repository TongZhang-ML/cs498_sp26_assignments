\begin{Q}
\textbf{Stable Diffusion 1.5 Fine-Tuning on Visible Prompt/Image Pairs. 50 Points}

In this problem, you will fine-tune \emph{Stable Diffusion 1.5} on a visible
set of generated prompt/image pairs.
The released data-generation pipeline creates:
\begin{itemize}
\item 100 visible training prompts,
\item for each prompt, 20 candidate images from Stable Diffusion 1.5,
\item a frozen visible reward model to pick the best candidate,
\item one saved visible training image for each prompt.
\end{itemize}
Students do \emph{not} implement the reward-filtering pipeline.
Instead, they receive the filtered visible prompt/image pairs and fine-tune on
them directly.

The visible files are
\[
\texttt{data/train.jsonl},\quad
\texttt{data/finetune\_images/}.
\]
Each visible row contains:
\begin{itemize}
\item \texttt{id}
\item \texttt{prompt}
\item \texttt{image\_path}
\end{itemize}

The released solution file also contains a separate visible evaluation set of
10 prompts that are similar to, but not identical with, the training prompts.
Students should:
\begin{itemize}
\item generate 10 base-model images on that evaluation set,
\item fine-tune the model on the visible training pairs,
\item generate 10 fine-tuned images on the same evaluation set,
\item print a LaTeX table containing the final training loss,
\item save the base and fine-tuned images,
\item save a 4-column comparison image with the left two columns from the base
model and the right two columns from the fine-tuned model.
\end{itemize}

You are provided with a starter Python file.
Your task is to fill in \emph{only} the functions whose docstring begins with
\textbf{Implement:}.
For clarity, the three graded implementation functions are:
\begin{align*}
&\texttt{build\_pipeline},\\
&\texttt{train\_diffusion\_batch},\\
&\texttt{generate\_prompt\_set}.
\end{align*}

\medskip
\noindent\textbf{Training protocol.}
The released code follows this procedure:
\begin{itemize}
\item load Stable Diffusion 1.5 from \texttt{diffusers},
\item freeze the VAE and text encoder,
\item fine-tune the UNet on the visible prompt/image pairs,
\item shuffle the visible rows each epoch,
\item train with one fixed default configuration,
\item save visible base and fine-tuned images on the 10 evaluation prompts.
\end{itemize}

\medskip
\noindent\textbf{Visible report.}
Students must report:
\begin{itemize}
\item a LaTeX table containing the final training loss,
\item the 10 base-model evaluation images,
\item the 10 fine-tuned evaluation images,
\item the 4-column comparison figure,
\item the base-model evaluation score from \texttt{diffusion\_eval.py},
\item the fine-tuned evaluation score from \texttt{diffusion\_eval.py}.
\end{itemize}

\medskip
\noindent\textbf{Autograder.}
Gradescope will:
\begin{itemize}
\item run unit tests on the three core functions above,
\item run \texttt{python diffusion\_solution.py},
\item verify that the base and fine-tuned evaluation images are produced,
\item score the saved images with \texttt{python diffusion\_eval.py} on the
visible evaluation prompts.
\end{itemize}
The score breakdown is:
\begin{itemize}
\item 36 points for correct implementations of the three core functions
(12 points each),
\item 8 points for the final report and required pictures,
\item 6 points for the evaluation score of the saved fine-tuned images.
\end{itemize}

\begin{enumerate}

\item \textbf{\texttt{build\_pipeline}: build the fine-tuning pipeline (12 points).}\\
Implement \texttt{build\_pipeline(...)}.
It should load Stable Diffusion 1.5, move it to the target device, freeze the
VAE and text encoder, and leave the UNet trainable.

\item \textbf{\texttt{train\_diffusion\_batch}: one diffusion training step (12 points).}\\
Implement \texttt{train\_diffusion\_batch(...)}.
For one batch of visible prompt/image rows, it should load the saved images,
encode them into latents, sample diffusion noise and timesteps, and run one
UNet update step with the diffusion loss.

\item \textbf{\texttt{generate\_prompt\_set}: generate one image for each evaluation prompt (12 points).}\\
Implement \texttt{generate\_prompt\_set(...)}.
It should deterministically generate one image for each prompt in the provided
evaluation set, using the configured seeds and generation hyperparameters.

\item \textbf{Final Reporting and Picture Submission (8 points).}\\
Run the released training script and include:
\begin{itemize}
\item the LaTeX loss table,
\item the 10 base-model evaluation images,
\item the 10 fine-tuned evaluation images,
\item the saved comparison image.
\end{itemize}

\item \textbf{Evaluation Score (6 points).}\\
Run \texttt{python diffusion\_eval.py} on the saved fine-tuned evaluation
images and report the resulting score.
\end{enumerate}

\begin{solution}

Reference numbers depend on the installed SD 1.5 checkpoint, GPU, and runtime
environment.
For TA/reference grading on the visible 10-prompt evaluation set:
\begin{itemize}
\item a base-model score around \texttt{4.0} is reasonable,
\item a solid fine-tuned score around \texttt{5.4} is a good target,
\item grading should use a tolerance band rather than exact equality.
\end{itemize}
