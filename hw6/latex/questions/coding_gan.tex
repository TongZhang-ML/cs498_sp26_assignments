\begin{Q}
\textbf{Class-Conditioned GAN in a Pretrained MNIST VAE Latent Space. 50 Points}

In this problem, you will train a simple \emph{class-conditioned GAN} on the 16-dimensional latent space of a pretrained unconditional VAE for MNIST.
You will then decode generated latent points back into MNIST images using the provided VAE decoder.

This keeps the GAN small and fast enough for local training while still testing whether you can implement conditional adversarial learning and distribution matching.

The visible files are
\[
\texttt{data/train\_latents.pt},\quad
\texttt{data/mnist\_vae.pt}.
\]
The hidden autograder file is \texttt{data/test\_latents.pt}. It is \emph{not} released to students.
The visible latent file contains:
\begin{itemize}
\item \texttt{latents}: a tensor of shape $(N, 16)$,
\item \texttt{labels}: a tensor of shape $(N,)$ with MNIST class labels in $\{0,\dots,9\}$.
\end{itemize}
The file \texttt{mnist\_vae.pt} stores the pretrained unconditional VAE checkpoint used by your starter code.

You are provided with a starter Python file.
Your task is to fill in \emph{only} the functions whose docstring begins with \textbf{Implement:}.
For clarity, the four graded functions are:
\begin{align*}
&\texttt{build\_gan\_models},\quad
\texttt{generator\_step\_loss},\\
&\texttt{discriminator\_step\_loss},\quad
\texttt{train\_gan\_batch}.
\end{align*}

\medskip
\noindent\textbf{Setup.}
The intended pipeline is:
\begin{itemize}
\item load the 16D latent train split,
\item build a class-conditioned generator $G(z, y)$ that maps Gaussian noise and a digit label to a latent vector,
\item build a class-conditioned discriminator $D(h, y)$ that scores whether a latent vector $h$ matches the requested class label $y$,
\item train the GAN in latent space,
\item generate exactly 1{,}000 latent samples for each class,
\item decode one generated latent per class with the provided VAE decoder for reporting.
\end{itemize}

\medskip
\noindent\textbf{Why latent space?}
The provided unconditional VAE already compresses binarized MNIST images into a compact 16D representation.
This lets you focus on class-conditioned GAN training rather than on stabilizing pixel-level image generation.
The decoder in \texttt{mnist\_vae.pt} is unconditional, fixed, and should be used as-is.

\medskip
\noindent\textbf{Training and evaluation.}
The starter code will train one fixed class-conditioned GAN configuration and save:
\begin{itemize}
\item \texttt{outputs/gan\_generated\_latents.pt},
\item \texttt{outputs/gan\_samples.png}.
\end{itemize}
The generated latent file should contain exactly 10{,}000 samples:
\begin{itemize}
\item 1{,}000 generated latent points for class 0,
\item 1{,}000 generated latent points for class 1,
\item \dots
\item 1{,}000 generated latent points for class 9.
\end{itemize}
Locally, students should run only \texttt{--mode train}; this requires only the visible files above.
On Gradescope, the autograder will additionally provide hidden \texttt{data/test\_latents.pt}.
The hidden evaluation compares:
\begin{itemize}
\item a random stratified sample from the visible train latent set against the hidden test latent set,
\item the saved/generated 10{,}000 GAN latent samples against the hidden test latent set.
\end{itemize}
The metric is the Fr\'echet distance between two Gaussian fits in the 16D latent space:
\begin{itemize}
\item compute the mean and covariance of one latent set,
\item compute the mean and covariance of the other latent set,
\item measure how far the two Gaussians are from each other.
\end{itemize}
This is analogous to FID, except that it is applied directly to the 16-dimensional VAE latent vectors rather than to Inception image features.
Lower Fr\'echet distance is better, and hidden-test grading will depend on this value.

\begin{enumerate}

\item \textbf{\texttt{build\_gan\_models}: build the conditional generator and discriminator (10 points).}
Implement \texttt{build\_gan\_models(device, hidden\_dim)}.
It should construct a simple class-conditioned generator and discriminator, move them to the requested device, and return them.

\item \textbf{\texttt{generator\_step\_loss}: generator objective (10 points).}
Implement \texttt{generator\_step\_loss(...)}.
It should:
\begin{itemize}
\item sample class-conditioned fake latent points,
\item run them through the discriminator,
\item compute the non-saturating generator loss.
\end{itemize}

\item \textbf{\texttt{discriminator\_step\_loss}: discriminator objective (10 points).}
Implement \texttt{discriminator\_step\_loss(...)}.
It should compute the discriminator loss on:
\begin{itemize}
\item real latent points from the provided split,
\item fake latent points from the generator,
\item the matching class labels.
\end{itemize}

\item \textbf{\texttt{train\_gan\_batch}: one alternating GAN update step (10 points).}
Implement \texttt{train\_gan\_batch(...)}.
It should perform exactly one batch update:
\begin{itemize}
\item update the discriminator on real and fake latent points,
\item update the generator against the discriminator,
\item return scalar batch metrics for logging.
\end{itemize}
The outer epoch loop in \texttt{train\_gan(...)} is provided.

\item \textbf{Training report and decoded samples (5 points).}
Run the script in \texttt{--mode train} and report:
\begin{itemize}
\item the final training losses printed by the script,
\item one decoded generated MNIST sample for each class from \texttt{outputs/gan\_samples.png}.
\end{itemize}
