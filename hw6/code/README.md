## Implementation

Install diffusion dependencies with:

`pip install -r requirements.txt`

Homework 6 contains two coding problems:

- `gan`: controllable generation by fine-tuning a pretrained StyleGAN with a small trainable control module
- `diffusion`: CLIP-score guided LoRA fine-tuning of Stable Diffusion 1.5

Both problems are designed in the same style as `hw3` and `hw5`:

- students implement only functions whose docstrings begin with `Implement:`
- visible train/val data is provided
- hidden test evaluation is performed from saved checkpoints
- the student release should come from `code/template/`

## gan

### Solution code

`gan/gan_solution.py`

### Data files

`gan/gan_gendata.py` should generate or package:

- `data/train.jsonl`
- `data/val.jsonl`
- `data/test.jsonl`

Optional visible helper files may include:

- pretrained latent codes
- frozen attribute labels
- a frozen attribute classifier
- a base StyleGAN checkpoint prepared by the course staff

Visible student data should include `train/val`.
The autograder may additionally provide hidden `test.jsonl` or hidden evaluation metadata.

### Intended training setup

This problem starts from a pretrained StyleGAN generator. Students should:

- load the pretrained generator
- freeze the backbone except for a lightweight controllable module
- train that module using attribute reward/classification loss plus preservation regularization
- save `outputs/gan_model.pt`

### Autograder behavior

- unit tests target functions whose docstrings begin with `Implement:`
- running the script in train mode saves `outputs/gan_model.pt`
- the checkpoint should contain only the lightweight trainable module plus minimal metadata
- running with `--mode test` loads that checkpoint and evaluates on hidden seeds/attributes with frozen hidden metrics

## diffusion

### Solution code

`diffusion/diffusion_solution.py`

### Data files

`diffusion/diffusion_gendata.py` should generate or package:

- `data/train.jsonl`
- `data/val.jsonl`
- `data/test.jsonl`

Optional visible helper files may include:

- prompt templates
- frozen reward-model metadata
- a frozen classifier for evaluation
- baseline SD-1.5 outputs for comparison

### Intended training setup

This problem starts from Stable Diffusion 1.5. Students should:

- use prompt splits generated by `diffusion_gendata.py`
- load SD 1.5 from `diffusers`
- freeze the base model except for UNet LoRA layers
- generate 8 candidate images per prompt using at least 20 inference steps
- score the candidates with a frozen CLIP model
- pick the top-1 candidate and do one LoRA-only diffusion update
- save `outputs/diffusion_model.pt` with LoRA weights only
- save a visible validation image sheet with 4 generations and their average score

### Autograder behavior

- unit tests target functions whose docstrings begin with `Implement:`
- running the script in train mode saves `outputs/diffusion_model.pt`
- the checkpoint should contain only the LoRA adapter plus minimal metadata
- running with `--mode test` loads the saved checkpoint and evaluates on one hidden prompt by averaging 4 generated-image scores
